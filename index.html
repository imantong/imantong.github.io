数组缓存

对于给定同样的参数总是返回同样的值的函数，可以在缓存中记录下计算结果，避免将来调用时重复计算。用同样的参数调用的时候，直接从缓存中查询结果来返回，而不用再重复计算。原始斐波那契递归算法中有很多重复计算的工作，可以利用一个数组把每次计算的结果缓存起来，避免重复计算，这样可以节省许多时间。

对象缓存

看有的博客说将上缓存类型由数组换成对象，效率要高一些。是因为数组在计算时，前面的项要初始化为undefined？？不太懂。。。

尾调用优化

函数调用时要保存调用位置和内部变量等信息，称为调用帧。函数嵌套调用会形成调用栈。递归容易导致内存问题，当调用达到可分配的上限，会造成栈溢出，尾调用可以解决这个问题。尾调用优化只保留内层函数的调用帧，可以节省内存。

闭包

发表于 2018-05-30

闭包是指有权访问另一个函数作用域中的变量的函数。
创建闭包的常见方式，就是在一个函数内部创建另一个函数。

题目：如果页面中有10个按钮，通过循环来给每个按钮绑定点击事件，则点击第3个按钮，输出什么？

输出都是10。
因为click事件是在for循环结束才触发的，此时变量i的值已经是10。每个事件函数引用的都是同一个全局变量i，所以在事件函数中顺着作用域链从内向外查找变量i时，找到的都是10.

解决：

  1. 使用IIFE（立即执行函数）把事件函数封闭起来，为每个迭代生成一个新的作用域，在每个迭代中储存i的值，返回事件函数。

改进：将i作为参数传递到IIFE中去，变量名可以任意取，比如叫j.

2.上面使用IIFE在每次迭代时都创建一个新的作用域，换句话说，每次迭代我们都需要一个块级作用域。可以用let声明变量。

for循环中，变量i是let声明的，当前的i只在本轮循环有效。每次迭代，变量都会重新声明，随后的每个迭代都会使用上一个迭代结束时的值来初始化这个变量。

数组去重

发表于 2018-05-30

unique([1,3,3,2]) //return [1,3,2]

unique([1,3,’3’,2]) //return [1,3,’3’,2]

1.普通版

时间复杂度为O(n^2)

定义一个变量数组res保存结果，遍历需要去重的数组，遍历res，如果该元素已经存在res中，则为重复元素，否则添加到res中。

2.简化版

时间复杂度还是O(n^2)

3.Hash结构

时间复杂度为O(n)

定义一个对象obj，将数组元素作为对象的键，遍历数组，如果键已经存在，则为重复元素，否则将元素存入res结果数组，并为obj添加键。

4.hasOwnProperty

通过对象的hasOwnProperty()方法检测给定的字符串是否是对象的自有属性。

5.区分对象类型

上面的typeof不能区分两个对象，比如{val:1}和{val:2}，typeof item + item都是”object[object Object]”，可以使用JSON.stringify将对象序列化。

6.ES6 Map

JavaScript的对象本质上是键值对的集合（Hash结构），但是只能用字符串作为键。
ES6的Map和对象类似，但“键”的范围不限于字符串，可以是各种类型的值。

7.ES6 Set

Set类似于数组，但是成员的值都是唯一的，没有重复的值。
