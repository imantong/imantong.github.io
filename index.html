git管理修改

发表于 2018-01-11

工作区和暂存区

工作区（working directory): 电脑里的目录

Git版本库（repository): 工作区中到.git目录

暂存区（stage): 在版本库中

master分支： git自动创建

HEAD指针： 指向master

git add: 把文件修改添加到暂存区

git commit： 把暂存区的所有内容提交到当前分支

管理修改

每次修改，如果不 add 到暂存区，就不会加入到 commit 中。

删除文件

一般，直接在文件管理器中删除文件，或在终端用 rm 命令删除。

然后，要从版本库中删除文件，要用 git rm，并且 git commit。

position属性

发表于 2018-01-05

position属性用于指定一个元素在文档中的定位方式。

取值：

  • static : 默认，即没有定位，元素出现在正常流中。
  • relative : 为元素预留空间，在文档中的正常位置偏移给定的值。
  • absolute : 不为元素预留空间，通过指定元素相对于最近的非 static 定位祖先元素的偏移，来确定元素位置。
  • fixed : 不为元素预留空间，元素的位置相对于浏览器窗口是固定位置。元素的位置在屏幕滚动时不会改变。
  • sticky : 粘性定位是相对定位和固定定位的混合。元素在跨越特定阈值前为相对定位，之后为固定定位。

重叠的元素：

z-index：指定了一个元素的堆叠顺序。一个元素可以有正数或负数的堆叠顺序，具有更高堆叠顺序的元素总是在较低的堆叠顺序元素的前面。

伪类选择器

发表于 2018-01-04

伪类选择器：

:first-child 父元素的第一个子元素

:last-child 父元素的最后一个子元素

:nth-child(an+b) 匹配文档树中在其之前具有 an+b-1 个兄弟节点的元素，其中 n 为正值或零值。简单点说，就是匹配那些在同系列兄弟节点中的位置与模式 an+b 匹配的元素。a 和 b 都必须为整数，并且元素的第一个子元素的下标为 1。

eg:

tr:nth-child(2n+1) 表示HTML表格中的奇数行。

tr:nth-child(odd) 表示HTML表格中的奇数行。

tr:nth-child(2n) 表示HTML表格中的偶数行。

tr:nth-child(even) 表示HTML表格中的偶数行。

:nth-of-type(an+b) 匹配那些在它之前有 an+b-1 个相同类型兄弟节点的元素，其中 n 为正值或零值。

初始化设置

因为Git是分布式版本控制系统，所以，每台机器都必须自报家门：你的名字和邮箱。

$ git config --global user.name "Your name"

$ git config --global user.email "email@example.com"

注意:git config --global命令中的--global参数，表示你这台电脑上所有的Git仓库都会使用这个配置。

Mac下使用Hexo+Github搭建博客

发表于 2017-12-27

前言

搭建博客有很多方法，这个博客就是用Github+Hexo的方法，这种方法上手简单，维护方便，界面美观大方，非常适合小白上手！

参考了知乎上很多大神的博客，终于搭建好了自己的博客。

写个博客记录一下，免得以后忘了。

PS：我的系统：macOS 10.13

搭建步骤

Github创建个人仓库

Github是一个代码托管和社交编程网站。

如果没有Github账号，登录Github注册账号:https://github.com，点击New repository创建新仓库，仓库名为：username.github.io，username是你的Github账号名，比如我的仓库名为：imantong.github.io

安装Git

Git是一个开源的分布式版本控制系统。

关于Git的教程：

安装过程

1.下载安装

Git官网下载地址：Git - Download for mac，可以下载最新版。我安装的是git 2.15.0。

打开安装包，我们需要把.pkg的安装包装到系统中。我双击后，显示无法打开。然后在网上百度，是由于这个需要权限，所以直接点击安装是无法成功的。

方法是按住control键之后，再点击pkg文件。这时会弹出安装程序的界面，然后选择打开，就可以完成安装。

安装好后，在mac终端输入git --version测试是否安装成功。

2.设置配置信息

终端输入：

git config --global user.name "你的Github用户名"

git config --global user.email "你的Github注册邮箱"

3.生成ssh密钥文件

Git关联远端仓库时候需要提供公钥，本地保存私钥，每次与远端仓库交互时候，远端仓库会用公钥来验证交互者身份。使用以下指令生成密钥。

在终端输入：cd ~，进入根目录；

输入命令：ssh-keygen，如果有提示，一直按回车。

4.将ssh key添加到Github

生成密钥后，在本地的/Users/当前电脑用户/.ssh目录下会生成两个文件id_rsa、id_rsa.pub，id_rsa文件保存的是私钥，保存于本地，id_rsa.pub文件保存的是公钥，需要将里面内容上传到远端仓库。

  • 输入cd指令，进入当前用户目录
  • 输入ls -a指令，查看当前用户目录下所有文件，包括隐藏文件
  • 输入cd .ssh指令，进入.ssh目录
  • 输入ls指令，查看.ssh目录下的文件
  • 输入cat id_rsa.pub指令，查看id_rsa.pub文件中内容

在个人设置页面，左边选中SSH and GPG keys，在右边添加公钥，title是key的名称，可以随便取，可更改，key是上面我们获取到的公钥，填写完毕后点击add SSH key按钮，这样远端就添加到了密钥。

linux基础命令：

  • sudo -s 获取绝对用户权限
  • cd xxx 进入xxx目录
  • ls (-a/-A) 显示当前路径下所有文件(隐藏的)
  • pwd 显示当前绝对路径
  • mkdir xxx 创建文件夹xxx
  • man xxx 查看xxx命令手册

安装Node.js

Hexo基于Node.js，Node.js下载地址：Download | Node.js 下载安装包，注意安装Node.js会包含环境变量及npm的安装。

安装后，检测Node.js是否安装成功，在命令行中输入 node -v

检测npm是否安装成功，在命令行中输入npm -v

至此，安装Hexo的环境已经全部搭建完成。

安装Hexo

Hexo就是我们的个人博客网站的框架。

这里需要在电脑里创建一个文件夹，可以命名为hexo，Hexo框架与以后你自己发布的网页都在这个文件夹中。

在终端进入hexo文件夹，使用npm命令安装Hexo，输入:

npm install -g hexo-cli

这里会报错：

npm ERR! Please try running this command again as root/Administrator.

输入：

sudo npm install -g hexo-cli

此时会要求输入Password，这里应该是你电脑的开机密码，注意光标是不会移动的，正常输入密码就好。

安装完成。

建站

Hexo安装完成后，输入

hexo init

注意，这里的命令都是作用在刚刚创建的hexo文件夹中

Hexo会新建所需要的文件。

然后输入

npm install

这里会报错，和上面的问题一样，输入

sudo hexo init

本地测试

输入：

hexo g

再输入：

hexo s

在浏览器地址栏输入：localhost:4000

可以看到搭建好的博客。

最后 control+C 关闭服务器。

常用的hexo命令

npm install hexo -g #安装Hexo

npm update hexo -g #升级

hexo init #初始化博客

命令简写

hexo n “我的博客” == hexo new “我的博客” #新建文章

hexo g == hexo generate #生成

hexo s == hexo server #启动服务预览

hexo d == hexo deploy #部署
