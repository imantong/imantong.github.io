“Mac下使用Hexo+Github搭建博客“

发表于 2017-12-27

前言

搭建博客有很多方法，这个博客就是用Github+Hexo的方法，这种方法上手简单，维护方便，界面美观大方，非常适合小白上手！

参考了知乎上很多大神的博客，终于搭建好了自己的博客。

写个博客记录一下，免得以后忘了。

PS：我的系统：macOS 10.13

搭建步骤

1.Github创建个人仓库

Github是一个代码托管和社交编程网站。

如果没有Github账号，登录Github注册账号:https://github.com，点击New repository创建新仓库，仓库名为：username.github.io，username是你的Github账号名，比如我的仓库名为：imantong.github.io

2.安装Git

Git是一个开源的分布式版本控制系统。

安装过程

1.下载安装

Git官网下载地址：Git - Download for mac，可以下载最新版。我安装的是git 2.15.0。

打开安装包，我们需要把.pkg的安装包装到系统中。我双击后，显示无法打开。然后在网上百度，是由于这个需要权限，所以直接点击安装是无法成功的。

方法是按住control键之后，再点击pkg文件。这时会弹出安装程序的界面，然后选择打开，就可以完成安装。

安装好后，在mac终端输入git --version测试是否安装成功。

2.设置配置信息

终端输入：

git config --global user.name "你的Github用户名"

git config --global user.email "你的Github注册邮箱"

3.生成ssh密钥文件

Git关联远端仓库时候需要提供公钥，本地保存私钥，每次与远端仓库交互时候，远端仓库会用公钥来验证交互者身份。使用以下指令生成密钥。

在终端输入：cd ~，进入根目录；

输入命令：ssh-keygen，如果有提示，一直按回车。

4.将ssh key添加到Github

生成密钥后，在本地的/Users/当前电脑用户/.ssh目录下会生成两个文件id_rsa、id_rsa.pub，id_rsa文件保存的是私钥，保存于本地，id_rsa.pub文件保存的是公钥，需要将里面内容上传到远端仓库。

  • 输入cd指令，进入当前用户目录
  • 输入ls -a指令，查看当前用户目录下所有文件，包括隐藏文件
  • 输入cd .ssh指令，进入.ssh目录
  • 输入ls指令，查看.ssh目录下的文件
  • 输入cat id_rsa.pub指令，查看id_rsa.pub文件中内容

在个人设置页面，左边选中SSH and GPG keys，在右边添加公钥，title是key的名称，可以随便取，可更改，key是上面我们获取到的公钥，填写完毕后点击add SSH key按钮，这样远端就添加到了密钥。

linux基础命令：

  • sudo -s 获取绝对用户权限
  • cd xxx 进入xxx目录
  • ls (-a/-A) 显示当前路径下所有文件(隐藏的)
  • pwd 显示当前绝对路径
  • mkdir xxx 创建文件夹xxx
  • man xxx 查看xxx命令手册

3.安装Node.js

Hexo基于Node.js，Node.js下载地址：Download | Node.js 下载安装包，注意安装Node.js会包含环境变量及npm的安装。

安装后，检测Node.js是否安装成功，在命令行中输入 node -v

检测npm是否安装成功，在命令行中输入npm -v

至此，安装Hexo的环境已经全部搭建完成。

4.安装Hexo

Hexo就是我们的个人博客网站的框架。

这里需要在电脑里创建一个文件夹，可以命名为hexo，Hexo框架与以后你自己发布的网页都在这个文件夹中。

在终端进入hexo文件夹，使用npm命令安装Hexo，输入:

npm install -g hexo-cli

这里会报错：

npm ERR! Please try running this command again as root/Administrator.

输入：

sudo npm install -g hexo-cli

此时会要求输入Password，这里应该是你电脑的开机密码，注意光标是不会移动的，正常输入密码就好。

安装完成。

5.建站

Hexo安装完成后，输入

hexo init

注意，这里的命令都是作用在刚刚创建的hexo文件夹中

Hexo会新建所需要的文件。

然后输入

npm install

这里会报错，和上面的问题一样，输入

sudo hexo init

6.本地测试

输入：

hexo g

再输入：

hexo s

在浏览器地址栏输入：localhost:4000

可以看到搭建好的博客。

最后 control+C 关闭服务器。

常用的hexo命令

npm install hexo -g #安装Hexo

npm update hexo -g #升级

hexo init #初始化博客

命令简写

hexo n “我的博客” == hexo new “我的博客” #新建文章

hexo g == hexo generate #生成

hexo s == hexo server #启动服务预览

hexo d == hexo deploy #部署

8.更换主题

如果你不喜欢Hexo默认的主题，可以更换不同的主题。

主题传送门：Themes

现在把默认主题更改成Next主题。

在终端输入(在hexo目录下)：

git clone https://github.com/iissnan/hexo-theme-next themes/next

这是将Next主题下载到blog目录的themes主题下的next文件夹中。

打开站点的_config.yml配置文件，修改主题为next。

theme: next

打开主题配置文件，找到Scheme Settings，可以设置不同的样式（只需要把行首的#去除，#是注释）。

选好后，输入hexo g、hexo d，再次部署网站。

9.个性化设置

所谓个性化设置就是根据个人需要添加不同的插件及功能。

基本设置：

在站点配置文件修改站点基本信息

增加侧栏菜单条目

修改主题配置文件里的Menu Settings中的menu和menu_icons.

其中menu里是配置菜单项对应的页面位置（如：/love），menu_icons对应菜单项的图标，这里的图标是来自于Font Awesome ，所以你需要在Font Awesome网站上找到你需要的icon，然后把该icon的名字写在menu_icons对应菜单名后面。

注意冒号有一个英文输入状态的空格。

设置好后，在命令行里输入：

hexo new page "你所要增加的菜单项名称（要和你在menu中的填写要匹配）"

新建的页面在博客根目录下的source文件里，这时你就可以对新建的页面自定义设计。

10.发表博文

新建博文

打开终端，cd到对应文件夹，例如我是hexo，输入：

hexo n '文章题目‘

然后在hexo/source/_posts中就可以找到刚才新建的博文。

当然可以在 Hexo/source/_posts 中直接新建 .md 文件，效果是一样的。

新建页面

输入：

hexo new page '页面名称'

然后在 /Hexo/source 中就可以看到刚才新建的页面。

11.图床

当博文有图片时，如果直接放在本地文件夹中，会使加载速度变慢，可以把图片上传到某一网站，然后获得外部链接来插入图片。这种网站就被称为图床。

常见图床：

  • 七牛云
  • 贴图库
