git管理修改

发表于 2018-01-11

工作区和暂存区

工作区（working directory): 电脑里的目录

Git版本库（repository): 工作区中到.git目录

暂存区（stage): 在版本库中

master分支： git自动创建

HEAD指针： 指向master

git add: 把文件修改添加到暂存区

git commit： 把暂存区的所有内容提交到当前分支

管理修改

每次修改，如果不 add 到暂存区，就不会加入到 commit 中。

删除文件

一般，直接在文件管理器中删除文件，或在终端用 rm 命令删除。

然后，要从版本库中删除文件，要用 git rm，并且 git commit。

position属性

发表于 2018-01-05

position属性用于指定一个元素在文档中的定位方式。

取值：

  • static : 默认，元素出现在正常流中，没有也不接受位置属性设置 （top、right、bottom、left属性值设置）。

  • relative : 为元素预留空间，让元素从默认位置移动。

  • absolute : 不为元素预留空间，绝对定位元素会脱离文档流。绝对定位元素需要至少一个祖先元素设置了相对定位（绝对定位），不然元素定位会相对于页面的主体进行定位。

  • fixed : 不为元素预留空间，定位是相对于浏览器窗口，并且不会随滚动条进行滚动。

  • sticky : 粘性定位是相对定位和固定定位的混合。元素在跨越特定阈值前为相对定位，之后为固定定位。

重叠的元素：

z-index：指定了一个元素的堆叠顺序。一个元素可以有正数或负数的堆叠顺序，具有更高堆叠顺序的元素总是在较低的堆叠顺序元素的前面。

伪类选择器

发表于 2018-01-04

伪类选择器：

:first-child 父元素的第一个子元素

:last-child 父元素的最后一个子元素

:nth-child(an+b) 匹配文档树中在其之前具有 an+b-1 个兄弟节点的元素，其中 n 为正值或零值。简单点说，就是匹配那些在同系列兄弟节点中的位置与模式 an+b 匹配的元素。a 和 b 都必须为整数，并且元素的第一个子元素的下标为 1。

eg:

tr:nth-child(2n+1) 表示HTML表格中的奇数行。

tr:nth-child(odd) 表示HTML表格中的奇数行。

tr:nth-child(2n) 表示HTML表格中的偶数行。

tr:nth-child(even) 表示HTML表格中的偶数行。

:nth-of-type(an+b) 匹配那些在它之前有 an+b-1 个相同类型兄弟节点的元素，其中 n 为正值或零值。

初始化设置

因为Git是分布式版本控制系统，所以，每台机器都必须自报家门：你的名字和邮箱。

$ git config --global user.name "Your name"

$ git config --global user.email "email@example.com"

注意:git config --global命令中的--global参数，表示你这台电脑上所有的Git仓库都会使用这个配置。
